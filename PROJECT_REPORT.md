# Project Report: HireReady AI Interview Bot

## 1. Executive Summary
**HireReady AI** is an advanced, full-stack AI-powered interview practice platform designed to simulate realistic technical interviews. It leverages state-of-the-art Large Language Models (LLMs) to provide dynamic questioning, real-time feedback, and comprehensive performance analytics.

## 2. Technical Architecture
The project is built using a modern Monorepo structure, separating the concerns of the user interface and the processing engine.

### Frontend
*   **Framework:** Angular 21 (TypeScript)
*   **Key Features:** 
    *   Responsive dashboard for topic selection.
    *   Live camera and audio integration for session recording.
    *   Real-time transcription using Web Speech API.
    *   Dynamic interview navigation (Next Question/Finish).

### Backend
*   **Environment:** Node.js / Express
*   **AI Orchestration:** Langchain
*   **Database:** MongoDB (via Mongoose)
*   **File Handling:** Multer (for resume and audio uploads)

## 3. AI & RAG Implementation
This project stands out by using a sophisticated AI pipeline:

*   **Core LLM:** Powered by **Groq** using the **Llama 3.3-70b-versatile** model, delivering ultra-low latency responses with high reasoning capability.
*   **RAG (Retrieval-Augmented Generation):**
    *   Uses **Xenova/all-MiniLM-L6-v2** for local vector embeddings.
    *   When a user uploads a resume, the system splits the text into chunks and performs a similarity search to generate tailored interview questions based on the candidate's specific experience.
*   **Evaluation Engine:** Analyzes transcripts to provide a score (1-10) and detailed feedback across metrics like Technical Knowledge, Communication, and Confidence.

## 4. Key Features Delivered
1.  **Topic-Based Interviews:** Users can select specific domains (Frontend, Data Science, etc.) for targeted practice.
2.  **Resume-Based Interviews:** AI analyzes uploaded PDFs/DOCXs to ask personalized questions.
3.  **Real-time Evaluation:** High-speed scoring and feedback powered by Groq.
4.  **Secure Environment:** Implementation of `.env` protection and Git security best practices.
